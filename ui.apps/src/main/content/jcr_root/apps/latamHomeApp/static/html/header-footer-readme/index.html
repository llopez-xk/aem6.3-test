Overview

Below is the technical description to implement the Header component for LAN/TAM in any application outsite AEM

Installation and dependencies

In order to implement the Header Component, the user should follow a few steps to sucefully install the component in other pages than the ones on AEM.
The Header Component has been writen entirely in Angular Directives, this technique let us self contain all the assets/logic/markup for a component, creating an unique package that can be used everywhere. but the user will need to include the next libraries in order to work properly.

External Javascript Libraries

  • Modernzr
  • Jquery
  • Angular
  • Angular IE8 build
  • Angular Animate
  • Angular Sanitize
  • Angular Cookies

Header javascript libraries

  • app.js
  • dataLogin.js
  • header.js
  • nav.js
  • countrySelector.js
  • footer.js
  • login.js
  • HeaderCtrl.js

Styles

  • bootstrap.min.css
  • lan.css or tam.css, these files are defined in the LATAM Bootstrap framework
  • footer-lan.css or footer-tam.css
  • header-lan.css or header-tam.css

Below is an example of how to include these files

            
              <!DOCTYPE html>
              <html lang="en">
              <head>

                <link rel="stylesheet" href="css/bootstrap.min.css">
                <link rel="stylesheet" href="css/header-tam.css">

                <script src="scripts/modernzr.js"></script>
              </head>

              <body>
                <header implementation goes here>

                ... your app code

                <footer implementation goes here>

                <script src="/static/latam/scripts/external/jquery/jquery-1.11.1.min.js"></script>

                <!-- Angular -->
                <!--[if lt IE 9]> <script src="bower_components/angularjs-ie8-build/dist/angular.min.js"></script> <![endif]-->
                <!--[if gt IE 8]><!--><script src="bower_components/angular/angular.min.js" type="text/javascript"></script> <!-->

                <script src="/static/latam/scripts/external/angular-cookies/angular-cookies.min.js"></script>

                <script src="bower_components/angular-animate/angular-animate.min.js"></script>
                <script src="bower_components/angular-sanitize/angular-sanitize.min.js"></script>

                <script src="scripts/app/app.js"></script>
                <script src="scripts/app/services/dataLoad.js"></script>
                <script src="scripts/app/directives/header.js"></script>
                <script src="scripts/app/directives/nav.js"></script>
                <script src="scripts/app/directives/countrySelector.js"></script>
                <script src="scripts/app/directives/footer.js"></script>
                <script src="scripts/app/directives/login.js"></script>
                <script src="scripts/app/directives/HeaderCtrl.js"></script>

                </body>
              </html>

It is extremely necessary that you inject these libraries into your application, otherwise, the header will not work




Implementation

The Footer / Header components are self contained directives as said before, so they act like tags that will be replaced with the appropiated markup and styles
in this case, the tag element for header and footer have a few variables (that are used as an html tag attributes) that will be used as a "configuration" for the component itself

Header implementation

The header implementation should be with the <header></header> tag with a class 'Header' and the attribute header as below

            
              <header class="Header" header>

This tag have some required attributes that you have to pass in order to instantate the Header component. here is the full list of parameters

Parameter Description Required
header Tell Angular to inject the header subcomponents inside this tag YES
template-header Set the header container url template, for this demo, the template is in 'scripts/app/views/header.html', but the AEM backend team will provide this URL YES
template-nav Set the nav template, for this demo, the template is in 'scripts/app/views/nav.html', but the AEM backend team will provide this URL YES
template-countrySelector Set the countrySelector template, for this demo, the template is in 'scripts/app/views/countryselector.html', but the AEM backend team will provide this URL YES
template-login Set the login template, for this demo, the template is in 'scripts/app/views/login.html', but the AEM backend team will provide this URL YES
role Indicate the ROLE for this component, for accesibilty purpose NO
logo An URL for the logo that will be used on mobile version, in this case the logo image is in 'assets/images/content-image/header/logo_lancom.png' YES
logo-desktop An URL for the logo that will be used on desktop version, in this case the logo image is in 'assets/images/content-image/header/lancom.png' YES
inverse true or false, indicates if you want to use the 'Paginas fijas' version or the 'Home Header' version NO
tam true if you want to use the Header with styles for TAM (you have to include the header-tam.css file) NO
process true if you want to use the 'Header de procesos' version of the component NO
ng-controller Indicates what will be the controller for the template. should be set to HeaderCtrl YES

An example of the implementation can be: what this do is to instante the Header component with the TAM styles and the 'Paginas Fijas' look & feel 

            
              <header
                template-header="scripts/app/views/header.html"
                template-nav="scripts/app/views/nav.html"
                template-countryselector="scripts/app/views/countryselector.html"
                template-login="scripts/app/views/login.html"
                logo="assets/images/content-image/header/lancom.png"
                logo-desktop="assets/images/content-image/header/logo_lancom.png"
                ng-controller="HeaderCtrl"
                inverse="true"
                tam="true"
              ></header>

Footer implementation

The footer implementation should be with the <div></div> tag with a class 'FooterComponent' and the attribute footer as below

            
              <div footer class="FooterComponent">

The footer also have a few parameters that setup the footer depending on what you will need for your application.

Parameter Description Required
footer Tell Angular to inject the footer subcomponents inside this tag YES
template-url Tell Angular where to find the markup for the footer, this URL will be provided by the AEM team YES
tam true if you want to load the elements for TAM NO
process true if you want to load 'Footer procesos' version NO
payment true if you want to include the payment markup on the footer NO

Here is an example of how to include the complete version of the footer for TAM

            
              <footer
                template-url="scripts/app/views/footer.html"
                tam="true"
                payment="true"
              ></footer>
